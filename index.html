PRJ-001 · professional

HareHare

live

A devotional AI app — chat for spiritual guidance and scripture, learn how to perform a puja, and book a pandit for the real ceremony. Answers grounded in scripture, never invented.

FastAPIRAG Vertex AIReact / Ionic Firebase
visit live →
what it is

A Hindu devotional app. People chat with it for emotional support, scripture Q&A, and step-by-step puja how-tos — and when they're ready, book a pandit and pay for a real puja, end to end. Live in production, phone-OTP accounts, multilingual.

my role

I built the mobile app, the API server, and the agentic RAG engine from scratch, and designed the booking flow. The most complex system I've shipped.

how it works

Agentic RAG: an intent router picks the path, hybrid retrieval + cross-encoder reranking find the evidence, and an abstain-on-weak-evidence gate means it never invents scripture. Four services on GCP.

stack

FastAPI · Gemini 2.5 Flash · Vertex AI · Firebase (Auth · Firestore · Remote Config · App Check) · React · Ionic / Capacitor · Next.js · Razorpay · Cloud Run

PRJ-003 · automation

Notion → Docs

built

An automation that mirrors a Notion page — text, formatting, and images — into a clean Google Doc you can export as a correct PDF.

PythonNotion API Google Docs / Drive
problem

Notion's PDF export drops images and breaks formatting. Rebuilding pages by hand doesn't scale.

what I built

A Python script that reads a Notion page and rebuilds it faithfully in Google Docs — no copy-paste.

architecture
flowchart LR
    O2["Google OAuth2
cached + silent refresh"] --> ORCH
    N["Notion API
walk block tree"] --> ORCH["Python orchestrator"]
    ORCH --> CM["Normalize → internal content model
headings · lists · quotes · toggles · code · img"]
    CM -->|"text + paragraph styles"| DOCS["Google Docs API
single batched batchUpdate"]
    CM -->|images| IMG
    subgraph IMG["Image pipeline · public-URL constraint"]
        DL["download"] --> UP["Drive upload"] --> PUB["make public"] --> EMB["insertInlineImage"] --> DEL["delete Drive copy"]
    end
    EMB --> DOCS
    DOCS -->|"reverse-order insert"| PDF["Formatted Doc → PDF"]
highlight

Docs only embeds images by public URL, so Drive is a throwaway host; blocks insert in reverse order so the doc reads top-down.

stack

Python · Notion API · Google Docs & Drive API · OAuth 2.0
